# Capability Map

This document tracks the capabilities this lab is researching. It should change as prototypes reveal what matters.

## Priority Capabilities

| Capability | Why It Matters | Prototype Direction |
| --- | --- | --- |
| Long-term memory | Trust depends on recall that is useful, current, and erasable. | Test memory write, retrieval, correction, deletion, and explanation loops. |
| Life-goal coaching | A friend-like companion should remember what the user is trying to do. | Build diary-to-plan and plan-to-accountability loops. |
| Digital diary | Memory needs raw material, and users need a safe place to think. | Build daily reflection capture, summaries, and goal extraction. |
| Two-way conversation | The companion must feel responsive, not like a journaling form. | Measure turn-taking quality, repair behavior, and conversational memory. |
| Storytelling | Stories can help users reflect, reframe, and navigate the world. | Generate short personal stories based on diary context and user goals. |
| 3D embodiment | Presence may be part of what makes a companion feel friend-like. | Test a constrained avatar that expresses attention, state, and continuity. |

## Secondary Capabilities

| Capability | Why It Matters | Prototype Direction |
| --- | --- | --- |
| Audio conversation | Voice is a natural interface for companionship, but low-level voice infrastructure is increasingly commoditized. | Integrate existing speech APIs before building custom voice systems. |
| Music and voice | Sound can carry pacing, ritual, and identity. | Test voice styles, ambient loops, and musical transitions during reflection. |
| Internet research | Users need current information and help exploring the world. | Add source-aware retrieval inside conversation. |
| Email check-ins | The companion may need light re-engagement outside the app. | Test opt-in weekly summaries or accountability nudges. |
| Custom companions | Personalization may increase usefulness and identity. | Defer until base companion UX is compelling. |
| 3D environment control | A companion world can make memory, state, and presence visible. | Keep the environment tiny until avatar presence shows signal. |

## Research Risks

- Memory quality: bad recall, stale facts, and privacy mistakes can destroy trust.
- Wedge risk: "better companion" is too broad; each prototype needs a concrete job-to-be-done.
- Retention risk: novelty may produce demos but not repeated use.
- 3D pipeline: avatar work can become expensive and slow unless the first tests stay narrow.
- Cost: real-time voice, memory retrieval, and 3D rendering can become too expensive before product-market fit.
