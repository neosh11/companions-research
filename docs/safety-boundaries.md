# Product Boundaries

This lab researches AI companions as consumer software. The public research posture should stay clear, practical, and product-oriented.

## Hard Boundaries

- No minors.
- No medical advice.
- No financial advice.
- No autonomous real-world action without user approval.
- No claims that the companion is human, conscious, or alive.
- No hidden retention of user memory after deletion.

## Product Principles

- The companion should be honest about being AI.
- The user should be able to erase memory.
- The user should stay in control of outbound actions.
- Sensitive user information should be handled with extra care.
- The system should prefer clear consent over clever automation.

## Research Questions

- What makes a companion feel friend-like across sessions?
- What memory controls are simple enough for consumers but strong enough for trust?
- Which proactive behaviors are useful, and which feel intrusive?
- How should the companion explain what it remembers?
- How should the product separate diary, coaching, research, and entertainment modes?
