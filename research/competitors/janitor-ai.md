# Janitor AI

## Snapshot

- Category: AI character chat / roleplay platform
- Company: Janitor AI
- Website: https://janitorai.com/
- Status: Active consumer platform; official details need direct account-level verification
- Last researched: 2026-06-21

## Product Thesis

Janitor AI's thesis is flexible character chat with user-created bots, customizable scenarios, and fewer restrictions than mainstream character platforms.

## Core Interaction Loop

Users browse or create characters, choose a model/backend path, write roleplay chats, and use chat memory or lore to keep scenarios consistent.

## Job-To-Be-Done

Offer configurable character roleplay for users who want more control than Character.AI provides.

## Memory Model

Community posts discuss chat memory as a place to store lore, relationships, traits, and facts the bot should not forget. Memory quality depends on model/backend and user setup.

## Voice, Avatar, and Embodiment

Janitor AI is mostly text and character-card led. Voice and rich embodiment are not the main differentiators.

## Retention Loop

Retention comes from user-created characters, saved chats, lower filtering, and investment in lore/memory setup.

## Pricing and Business Model

Third-party 2026 reviews describe a free tier and paid/pro options, with some users also bringing external model/API costs. Official pricing should be verified from a logged-in account.

## What Users Love

Users value control, flexible character creation, less restrictive roleplay, and the ability to shape memory/lore.

## What Users Complain About

Complaints include setup friction, variable model quality, downtime/stability, and possible API costs.

## Strategic Lessons

Janitor AI shows that power users will tolerate friction for control. Companions should not require that friction for mainstream use, but can learn from explicit memory/lore editing.
