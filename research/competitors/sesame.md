# Sesame

## Snapshot

- Category: Personal agents / conversational voice / AI hardware
- Company: Sesame
- Website: https://www.sesame.com/
- Status: Active preview product and research; eyewear announced as coming 2027
- Last researched: 2026-06-21

## Product Thesis

Sesame's thesis is that natural conversational voice plus always-available personal agents can create a more fluid companion/assistant experience, eventually supported by wearable hardware.

## Core Interaction Loop

Users talk to personal agents, think out loud, follow threads, discover information, and eventually use hands-free eyewear.

## Job-To-Be-Done

Make voice agents useful for in-between moments where typing or app navigation would break flow.

## Memory Model

Public pages emphasize conversation history and natural speech. Durable personal memory controls are not yet clear from public preview information.

## Voice, Avatar, and Embodiment

Voice is the main embodiment. Sesame's Conversational Speech Model frames voice as end-to-end multimodal generation using conversation history for natural speech.

## Retention Loop

Retention would come from low-friction voice access, agent continuity, personal threads, and eventually hardware habit loops.

## Pricing and Business Model

Preview access is public-facing; eyewear is announced for 2027. Pricing for consumer agents and hardware needs verification when launched.

## What Users Love

Early coverage praises voice naturalness and flow. The strongest signal is that voice can feel like a product wedge, not just an input mode.

## What Users Complain About

Open concerns include privacy, over-humanized voice, unclear memory controls, and unknown pricing/hardware adoption.

## Strategic Lessons

Sesame supports the idea that voice and presence matter. Companions should study it for conversational pacing and hands-free use, while keeping memory and agency explicit.

## Open Questions

- What memory does Sesame keep across sessions?
- How will eyewear change re-engagement?
- What is the cost of serving long real-time conversations?
