# Prototypes

Experimental code can live here before anything graduates to the product repository.

Prototype order:

- Memory write, retrieval, correction, and deletion experiment.
- Diary capture and summary.
- Diary-to-plan coaching loop.
- Accountability check-in loop.
- Constrained avatar presence test.
- Storytelling experiment.
- Voice conversation loop.
- Email check-in flow.
- Avatar expression controller.
- 3D room or companion environment.

Each prototype should document:

- The job-to-be-done.
- The research question.
- Setup instructions.
- What worked.
- What failed.
- Cost notes.
- Whether it should move toward product or be discarded.
